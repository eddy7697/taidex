# Taidex 台股看板工具 —— 設計文件

- 日期:2026-07-02
- 狀態:設計已確認,待實作

## 1. 專案願景

給投資新手(Vincent 與配偶)使用的台股洞察 / 看板工具。核心設計哲學:

> **使用門檻低,操作上限高。**

- 「門檻低」= 對終端使用者(投資新手)的 UX 要求:打開即用、一眼看懂、不需教學。
- 「上限高」= 工程面要求:架構可持續長功能,深度資訊採漸進式揭露,重度使用者能挖深。

### 功能藍圖(依優先序,逐版疊加)

1. **看盤 / 自選股追蹤**(第一版,本 spec 範圍)
2. 大盤與產業總覽
3. 條件選股 / 找機會
4. 持股損益追蹤

本文件只完整規劃**第一版:看盤 / 自選股**,但架構須為後續三項預留擴充空間。

## 2. 需求摘要

| 面向 | 決策 |
|------|------|
| 第一版核心 | 看盤 / 自選股追蹤 |
| 投資風格 | 混合(存股 / 波段 / 短線)→ 資訊密度需可切換、欄位可自訂 |
| 即時性 | 盤中每分鐘更新即可(不需券商 tick） |
| 裝置 | 手機 + 電腦,響應式,手機為主力 |
| 帳號 | LINE 登入;每位使用者各自獨立的自選股清單 |
| 成本 | 先全用免費資料源與現有基礎設施,好用再升級 |
| 部署 | 使用者既有的 GKE 叢集(容器化) |
| 資料庫 | 使用者既有的 Cloud SQL **MySQL 8** |

## 3. 架構(方案 A:單體 Next.js 全端)

一個 Next.js(App Router)容器,同時作為前端與後端 API;連既有 Cloud SQL MySQL 8;
一個 Kubernetes CronJob 每天收盤後拉當日行情入庫。

```
┌─────────────────────────────────────────────┐
│                   GKE 叢集                     │
│                                               │
│  ┌───────────────────────┐   ┌─────────────┐ │
│  │  Next.js Deployment    │   │ K8s CronJob │ │
│  │  ┌─────────┐┌────────┐ │   │  每日收盤後   │ │
│  │  │ 前端 UI  ││ API    │ │   │  拉當日行情   │ │
│  │  │(App     ││ Routes │ │   │  入 DB       │ │
│  │  │ Router) ││        │ │   └──────┬──────┘ │
│  │  └─────────┘└───┬────┘ │          │        │
│  │       quote-service     │          │        │
│  │       (資料源抽象層)     │          │        │
│  └───────────┬───────────┘          │        │
└──────────────┼──────────────────────┼─────────┘
               │                      │
      ┌────────┴────────┐    ┌────────┴─────────┐
      │  外部免費資料源   │    │ Cloud SQL MySQL 8 │
      │  (盤中報價)      │    │ (使用者/自選股/    │
      └─────────────────┘    │  行情快取)         │
                             └──────────────────┘
```

### 技術選型

| 層 | 選擇 | 理由 |
|----|------|------|
| 框架 | Next.js(App Router)+ TypeScript | 前後端一套語言,單一可部署容器,觀摩心智負擔低 |
| ORM | Prisma | MySQL 8 支援成熟,schema 版控清楚易讀 |
| DB | Cloud SQL MySQL 8(既有) | 使用者現成資源,不額外開 DB |
| 認證 | NextAuth(Auth.js)+ LINE Provider | LINE 登入在台灣最直覺,官方 provider 現成 |
| 樣式 | Tailwind CSS | 響應式快速開發,一套程式兩種佈局 |
| 圖表 | 輕量圖表庫(如 lightweight-charts / Recharts) | K 線與走勢線;細節頁使用 |
| 部署 | Docker image → GKE Deployment + CronJob | 符合既有基礎設施 |

### 演進路徑

當某功能(如選股引擎)運算變重,再把該塊從單體抽成獨立服務(演進成前後端分離),
不需重寫。v1 刻意保持單體以維持節奏。

## 4. 資料源(分層,全部先免費)

後端 `quote-service` 抽象層對前端只暴露「給我這些代號的報價 / 這檔的歷史」介面,
底層資料源、快取策略、盤中盤後切換全部封裝其中。**換源 / 加源 / 加欄位不動前端。**

| 需求 | 來源 | 說明 |
|------|------|------|
| 盤中即時報價(每分鐘) | 證交所 MIS(`mis.twse.com.tw`) | 官方盤中揭示,約 5–20 秒延遲,免費。後端 proxy + 快取 30–60 秒,解決 CORS 與限流 |
| 每日行情 / 歷史 K 線 | 證交所 & 櫃買 OpenAPI(`openapi.twse.com.tw`) | 收盤後開高低收量,免費穩定。CronJob 每日入庫 |
| 基本面 / 籌碼 / 財報(未來選股用) | FinMind 免費方案 | 殖利率、EPS、三大法人買賣超等,免費有速率限制 |

- **盤中 / 盤後判斷**:後端依台股交易時段(平日 09:00–13:30)決定回即時源或回 DB 當日收盤,前端不需感知。
- **快取**:盤中報價於 DB / 記憶體快取 30–60 秒,多使用者共用同一份,降低對外請求。

## 5. 資料模型(初版)

```
User
  id            (PK)
  lineUserId    (LINE sub,唯一)
  displayName
  avatarUrl
  createdAt

WatchlistItem            // 各使用者各自獨立的自選股
  id            (PK)
  userId        (FK -> User)
  stockSymbol   (如 "2330")
  sortOrder     (使用者自訂排序)
  createdAt
  (userId, stockSymbol) 唯一

Stock                    // 台股標的主檔(代號 -> 名稱等)
  symbol        (PK, 如 "2330")
  name          (如 "台積電")
  market        (上市 / 上櫃)
  industry

DailyQuote               // 每日收盤行情(CronJob 入庫)
  id            (PK)
  stockSymbol   (FK -> Stock)
  date
  open, high, low, close, volume
  (stockSymbol, date) 唯一

UserColumnPref           // 自選股欄位自訂偏好(對應混合風格)
  userId        (FK -> User)
  columns       (JSON:使用者選擇顯示的欄位)
```

盤中即時報價不落 `DailyQuote`,以短快取形式提供;收盤後由 CronJob 寫入當日 `DailyQuote`。

## 6. 第一版看盤畫面

**設計原則:一眼看懂 → 想深入再點開。** 預設極簡,深度資訊藏在點擊後的細節頁。

### 手機版(主力)

- 頂端:大盤加權指數一行(紅漲綠跌,台股慣例)。
- 自選股卡片列表:大字報價、漲跌額、漲跌幅、迷你當日走勢線、成交量。
- 頂端搜尋:輸入名稱(「台積電」)或代號(「2330」)皆可加入。
- 每分鐘自動刷新,顯示「更新於 hh:mm」。
- 長按 / 滑動卡片:排序、刪除、拖曳調整順序。
- 點卡片:進個股細節頁(K 線、基本資料、法人買賣超)。
- 底部分頁:[自選] [大盤] [選股] [持股]——後三者為未來功能占位。

### 電腦版

同一份程式,寬螢幕自動切換為**表格 + 右側細節**佈局:
左側自選股列表(更多欄位:成交、漲跌、幅度、量、殖利率),點列右側直接展開該股 K 線與細節,不換頁。

### 「上限高」的漸進式揭露

- 個股細節頁圖表可切換週期(日 / 週 / 月)、可疊加均線:新手看價格,老手調指標。
- 自選股欄位可自訂(存股者加「殖利率」,波段者加「5 日漲幅」),回應混合風格。
- 預設乾淨,深度需自行開啟。

## 7. 元件與職責邊界

| 單元 | 職責 | 依賴 |
|------|------|------|
| `quote-service` | 對外統一報價 / 歷史查詢介面;封裝資料源、快取、盤中盤後切換 | 外部資料源、DB 快取 |
| `auth`(NextAuth + LINE) | LINE 登入、session、使用者建檔 | User 表 |
| `watchlist` API | 自選股 CRUD、排序、欄位偏好 | WatchlistItem、UserColumnPref |
| `stock-search` | 名稱 / 代號搜尋標的 | Stock 表 |
| 每日行情 CronJob | 收盤後拉當日行情與標的主檔更新入庫 | 外部 OpenAPI、DailyQuote、Stock |
| 前端 UI | 響應式看盤畫面、卡片 / 表格佈局、細節頁 | 上述 API |

每個單元對外以明確介面溝通,可獨立理解與測試。

## 8. 錯誤處理

- 外部資料源失敗 / 逾時:回退至 DB 最近一筆收盤,前端標示「資料延遲 / 非即時」。
- 盤中限流:共用快取吸收多使用者請求;超限時降級為快取內資料。
- LINE 登入失敗:導回登入頁並顯示可理解的錯誤訊息。
- 搜尋不到標的:提示確認代號 / 名稱。

## 9. 測試策略

- `quote-service`:單元測試盤中 / 盤後切換、快取命中、資料源失敗回退(mock 外部源)。
- `watchlist` API:CRUD、排序、跨使用者隔離(A 看不到 B 的清單)。
- 認證流程:LINE callback → 使用者建檔 → session。
- 前端:關鍵互動(加入 / 刪除自選股、響應式佈局切換)以整合測試覆蓋。

## 10. 非目標(第一版明確不做,YAGNI)

- 秒級即時 tick / 五檔報價 / 券商 API 串接。
- 大盤產業總覽、條件選股、持股損益(後續版本)。
- 下單交易功能(純資訊工具,不涉及下單)。
- 多語系(先中文)。

## 11. 部署

- Dockerfile 建置 Next.js standalone image。
- GKE:1 個 Deployment(Next.js)+ 1 個 CronJob(每日行情)+ Secret(DB 連線、LINE、FinMind 金鑰)。
- 連既有 Cloud SQL MySQL 8(Cloud SQL Auth Proxy 或私有 IP)。
- Prisma migrate 於部署流程執行 schema 遷移。
